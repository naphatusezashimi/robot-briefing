## ภาพรวมโปรเจกต์

สร้างหุ่นยนต์ที่พูดคุยโต้ตอบด้วยเสียงแบบเรียลไทม์ ตั้งไว้ที่วิทยาลัยเพื่อตอบคำถาม
ผู้มาติดต่อ เช่น หลักสูตร การรับสมัคร กิจกรรม และบริการวิชาชีพ

**Flow การทำงาน 5 ขั้น:**
1. รับเสียงจากไมโครโฟน (Input)
2. แปลงเสียงเป็นข้อความภาษาไทย (Speech to Text)
3. คิดคำตอบด้วย AI โดยอ้างอิงฐานข้อมูลวิทยาลัย (AI + Database)
4. แปลงคำตอบเป็นเสียงพูดภาษาไทย (Text to Speech)
5. ตอบกลับทางลำโพง + แสดงบนจอ + ไฟ LED (Output) แล้ววนรับคำถามถัดไป

## เป้าหมายของโปรเจกต์
- แข่งสิ่งประดิษฐ์
- ใช้งานจริงที่วิทยาลัย
- เป็นโปรเจกต์จบ

## สิ่งสำคัญที่สุด: หุ่นต้องไม่ตอบมั่ว

นี่คือหัวใจของโปรเจกต์และเป็นจุดที่กรรมการจะเจาะ ระบบ AI **ต้องตอบจาก
ฐานข้อมูลวิทยาลัยที่ให้ไว้เท่านั้น** หากไม่มีข้อมูล ให้ตอบว่ายังไม่มีข้อมูล
และแนะนำให้ติดต่อเจ้าหน้าที่ ห้ามแต่งข้อมูลขึ้นเองเด็ดขาด

ต้องทำให้แน่ใจว่า prompt ที่ส่งให้ AI มีคำสั่งนี้ชัดเจน และข้อมูลวิทยาลัย
ถูกแนบไปในทุกครั้งที่ถาม

## โครงสร้างไฟล์ที่อยากได้

```
robot/
├── main.py              # วนลูปการทำงานหลัก เชื่อมทุกส่วนเข้าด้วยกัน
├── config.py            # ตั้งค่า เช่น ชื่อ env variable ของ API key
├── stt.py               # โมดูลแปลงเสียงเป็นข้อความ (Speech to Text)
├── ai.py                # โมดูลถาม AI พร้อมแนบข้อมูลวิทยาลัย + กันตอบมั่ว
├── tts.py               # โมดูลแปลงข้อความเป็นเสียง (Text to Speech)
├── output.py            # โมดูลแสดงผล: เล่นเสียง + หน้าจอ + ไฟ LED
├── database.py          # โหลด/ค้นข้อมูลจาก college_data.json
├── college_data.json    # ฐานข้อมูลวิทยาลัย (มีตัวอย่างให้แล้ว)
├── simulator.py         # ตัวจำลองให้รันบนคอมได้ก่อนซื้ออุปกรณ์
├── requirements.txt     # รายการไลบรารีที่ต้องติดตั้ง
└── README.md            # วิธีติดตั้งและรัน
```

## โหมดจำลอง (สำคัญ — ทำก่อนเป็นอันดับแรก)

ทีมยัง**ไม่ได้ซื้ออุปกรณ์** ดังนั้นทุกโมดูลต้องมี "โหมดจำลอง" ที่รันบน
คอมพิวเตอร์ธรรมดาได้ โดยไม่ต้องมีไมค์/ลำโพง/จอจริง:

- **แทนไมโครโฟน** → พิมพ์คำถามทางคีย์บอร์ดแทนการพูด
- **แทนลำโพง** → ปริ้นข้อความคำตอบออกหน้าจอ (ยังไม่ต้องเล่นเสียง)
- **แทนหน้าจอ touchscreen** → แสดงในหน้าต่างคอม หรือหน้าเว็บ
- **แทนไฟ LED** → ปริ้นบอกสถานะ เช่น `[LED] ฟ้า = กำลังฟัง`

ออกแบบให้สลับระหว่าง "โหมดจำลอง" กับ "โหมดฮาร์ดแวร์จริง" ได้ง่าย
(เช่น ตั้งค่าตัวแปรเดียวใน config.py) เพื่อให้พอซื้ออุปกรณ์มาแล้วเปลี่ยน
มาใช้ของจริงได้โดยไม่ต้องรื้อโค้ด

## แผนการทำงานแบ่งเป็นเฟส

ทำทีละเฟส อย่าข้าม ทดสอบให้ผ่านก่อนค่อยไปเฟสถัดไป

### เฟส 1 — โครงและโหมดจำลอง (ทำได้เลยไม่ต้องมีอุปกรณ์)
- วางโครงไฟล์ทั้งหมดตามโครงสร้างด้านบน
- ทำ database.py โหลด college_data.json ได้
- ทำ ai.py เชื่อม Claude API พร้อม prompt กันตอบมั่ว
- ทำ simulator.py ที่พิมพ์คำถาม → ได้คำตอบจาก AI (ยังไม่ใช้เสียง)
- **เป้าหมายเฟสนี้:** พิมพ์คำถามเรื่องวิทยาลัยแล้วได้คำตอบถูกต้องบนคอม

### เฟส 2 — เพิ่มเสียง (ยังจำลองได้)
- ทำ stt.py และ tts.py เชื่อม Google API
- ทดสอบด้วยไฟล์เสียงตัวอย่างก่อน (ยังไม่ต้องมีไมค์จริง)

### เฟส 3 — ต่อฮาร์ดแวร์จริง (เมื่อซื้ออุปกรณ์แล้ว)
- เปิดโหมดฮาร์ดแวร์ใน config.py
- ต่อไมค์ ลำโพง จอ ไฟ LED จริงเข้ากับ Raspberry Pi 5
- ทดสอบทั้งระบบ

### เฟส 4 — เก็บสถิติและปรับปรุง
- บันทึก log ทุกคำถาม-คำตอบลงไฟล์ (ใช้ทำสถิติตอนแข่ง/ทำเล่มจบ)
- เพิ่มปุ่มพิมพ์สำรองเผื่อระบบฟังเสียงผิด
