# Design Spec: หุ่นตอบพร้อมแผนผังบอกทาง + แก้หัวข้อจอแชท

**วันที่:** 2026-06-17
**สถานะ:** รออนุมัติ
**ขอบเขต:** เพิ่มฟีเจอร์ให้เว็บ "กล่องพูดได้" แนบ **รูปแผนผังวิทยาลัย** ในคำตอบเมื่อผู้ใช้ถามเรื่องตำแหน่ง/ทาง, เพิ่ม **ตารางรหัสอาคาร (legend)** ลงฐานข้อมูลให้หุ่นบอกชื่อ/เลขอาคารได้แม่นโดยไม่หลุดกรอบกันตอบมั่ว, และแก้บั๊ก **หัวข้อจอแชทตัดคำเสียรูป (P1)**

---

## ที่มา

รีวิว UI จากของจริง (เปิด `/screen` + `/face`) พบว่า:

- **จอใบหน้า** แข็งแรงอยู่แล้ว ทุกสถานะ (รอ/ฟัง/คิด/ตอบ/ผิดพลาด) สื่ออารมณ์ชัด — ไม่แตะ
- **จอแชท** มีบั๊กหัวข้อ: "วิทยาลัยเทคนิคท่าหลวงฯ" ตัดบรรทัดกลางคำเป็น "ท่าหล / วงฯ" เพราะแย่งพื้นที่แนวนอนกับโลโก้และป้ายสถานะ (เห็นชัด ดูเหมือนพัง)

ผู้ใช้ขอเพิ่มฟีเจอร์ใหม่: ให้หุ่น **โชว์รูปบอกทาง/ตำแหน่ง** เวลาผู้มาติดต่อถามว่า "อยู่ตรงไหน / ไปยังไง" และส่งทรัพยากรมาให้:

- **แผนผังวิทยาลัย:** `http://www.tl.ac.th/pictures/plan2568.jpg` (JPEG 1981×976 แนวนอน ~222KB, แผนผังสีสันสดใส อาคารใช้รหัสตัวอักษร A–R และตัวเลข 1–9, มีสนามฟุตบอล ถนนรอบบริเวณ)
- **ตารางรหัสอาคาร→ชื่อสถานที่** (legend) — ดูส่วน "ข้อมูล legend ที่จะเพิ่ม"

## การตัดสินใจหลัก (locked)

1. **ทริกเกอร์รูป: คีย์เวิร์ดฝั่งเซิร์ฟเวอร์ (deterministic) + ปุ่มชิป** — ไม่ให้ AI เป็นคนตัดสินว่าจะโชว์รูปไหม
   - ตัด "ให้ AI ตัดสิน" เพราะไม่แน่นอน (บางทีลืม/โชว์เกิน) และเพิ่มความเสี่ยง "มั่ว" ที่เป็นจุดที่กรรมการเจาะ
2. **แสดงผล: แผนผังทั้งใบในฟองคำตอบ + แตะเพื่อขยายเต็มจอ (lightbox)** — เพราะแผนผังละเอียดและเป็นแนวนอน อ่านยากถ้าเล็ก
   - ยัง **ไม่** ปักหมุด/ไฮไลต์รายอาคาร (ต้องใช้พิกัดรายจุดบนรูป = งานเยอะ) → เก็บเป็นเฟส 2
3. **ที่มารูป: ดาวน์โหลด `plan2568.jpg` มาเก็บ `robot/web/static/map.jpg`** — ไม่ลิงก์ตรงเว็บวิทยาลัย
   - ตัด "hotlink เว็บนอก" เพราะหุ่นจะพึ่งเน็ต/เว็บวิทยาลัย ถ้าเว็บล่ม/ช้า รูปหาย; อีกทั้งเว็บเป็น HTTP ล้วน
4. **legend อาคาร: ใส่ใน `college_data.json`** — `ai.py` ป้อน `college_data.json` ทั้งก้อนเข้า system prompt อยู่แล้ว (`format_for_prompt` = `json.dumps` ทั้ง dict) ดังนั้นเพิ่ม section ใหม่ = AI ใช้ได้ทันที **ไม่ต้องแก้ prompt** และยังอยู่ในกรอบ "ตอบจากฐานข้อมูลเท่านั้น"
5. **ไม่แตะ:** `ai.py`, `database.py`, `config.py`, `face.html`, `face.js`, โฟลเดอร์ `mockups/` — การเลือกรูปเป็น presentation layer ล้วน อยู่ที่ web เท่านั้น สมองเดิม (กฎกันตอบมั่ว) ไม่เปลี่ยน

## รายละเอียดการแก้

| ไฟล์ | สถานะ | ทำอะไร |
|------|-------|--------|
| `robot/web/static/map.jpg` | ใหม่ (asset) | ดาวน์โหลด `plan2568.jpg` มาเก็บ |
| `robot/web/wayfinding.py` | ใหม่ | ฟังก์ชันบริสุทธิ์ `wants_map(text: str) -> bool` เช็คคีย์เวิร์ดตำแหน่ง/ทาง — แยกออกมาเพื่อ unit test ง่ายและไม่ทำให้ `server.py` รก |
| `robot/web/server.py` | แก้ | ใน `/ask` หลังได้คำตอบ เรียก `wants_map(question)` ถ้า `True` ใส่ `image="/static/map.jpg"` ในผล JSON (`jsonify(answer=..., image=...)`) |
| `robot/web/static/screen.js` | แก้ | `ask()` อ่าน `d.image` → ส่งต่อให้ `addMsg`; `addMsg` รองรับ `opts.image` วาด `<img class="map-thumb">` ในฟอง; เพิ่ม lightbox (เปิด/ปิด) ตอนแตะรูป; ปุ่มชิป "แผนผังวิทยาลัย" → `showMap()` แสดงรูปในฟองทันที **โดยไม่เรียก `/ask`** |
| `robot/web/static/screen.html` | แก้ | (ก) แก้ header (P1); (ข) เพิ่มชิป "🗺️ แผนผังวิทยาลัย" เป็นชิปแรกใน `#quickQs`; (ค) เพิ่ม markup + CSS ของ `.map-thumb` และ lightbox overlay |
| `robot/college_data.json` | แก้ | เพิ่ม section `"แผนผังและที่ตั้งอาคาร"` (legend + path รูป) — ดูด้านล่าง |
| `robot/web/test_server.py` | แก้ | เพิ่มเทสต์ `wants_map` (เคสเข้า/ไม่เข้า) + เทสต์ `/ask` ว่าแนบ `image` เมื่อถามตำแหน่ง และไม่มี `image` เมื่อถามเรื่องอื่น (ยัง mock `ask_ai` เหมือนเดิม) |

## พฤติกรรม / Data flow

```
ผู้ใช้ถาม (พูด/พิมพ์/แตะชิปคำถาม)
   → POST /ask {question}
   → ask_ai(question, college_data)        # สมองเดิม ตอบข้อความจากฐานข้อมูล (รวม legend)
   → wants_map(question)?                   # เช็คคีย์เวิร์ดฝั่ง server
        ใช่  → jsonify(answer, image="/static/map.jpg")
        ไม่  → jsonify(answer)
   → screen.js: addMsg("bot", answer, {image})   # มีรูป = วาด thumbnail แตะขยายได้

แตะชิป "แผนผังวิทยาลัย"
   → showMap()  →  addMsg("bot", "นี่คือแผนผังวิทยาลัยค่ะ แตะที่รูปเพื่อขยายดูได้เลย", {image:"/static/map.jpg"})
   (ไม่เรียก /ask, ไม่เปลือง quota, ตอบทันที)
```

**คีย์เวิร์ดตั้งต้นของ `wants_map`** (เจอคำใดคำหนึ่งใน question ถือว่าเข้าเงื่อนไข, เทียบแบบ substring):
`อยู่ที่ไหน, อยู่ไหน, ตรงไหน, ที่ไหน, ไปยังไง, ไปทางไหน, ไปยัง, เดินไป, ทางไป, เส้นทาง, แผนผัง, แผนที่, อาคาร, ตึก, ห้อง, สนาม, จุด, โรงอาหาร, หอประชุม, ป้อมยาม, เสาธง, ที่จอดรถ, บ้านพัก`
(ปรับเพิ่ม/ลดได้ตอน implement — เก็บเป็น list เดียวใน `wayfinding.py`)

## รายละเอียดการแก้หัวข้อ (P1)

ปัญหา: ใน `header` ของ `screen.html`, `.brand h1` (ฟอนต์ Mitr ~1.04rem) แย่งพื้นที่กับ `.logo` (48px) และ `.led` (ป้าย "พร้อมรับคำถาม") จนชื่อวิทยาลัยตัดบรรทัดกลางคำ

แนวแก้ (ปรับใน `screen.html` จุด CSS เป็นหลัก ไม่แตะ logic):
- ให้ `.brand` มี `min-width:0` (กัน flex item ดันล้น) — คงเดิม `flex` ที่ทำให้ยืดรับพื้นที่
- ลด `font-size` ของ `.brand h1` ลงเล็กน้อย และตั้ง `line-height` กระชับ + `text-wrap:balance` เพื่อถ้าต้องขึ้น 2 บรรทัดจะสมดุล ไม่เหลือ "วงฯ" ลอยบรรทัดเดียว
- ทำป้าย `.led` ให้กระชับลง (ลดข้อความ/ขนาดตัวอักษร หรือบนจอแคบเหลือจุด + ข้อความสั้น) เพื่อคืนพื้นที่ให้ชื่อ
- เป้าหมาย: "วิทยาลัยเทคนิคท่าหลวงฯ" อยู่บรรทัดเดียว (หรือ 2 บรรทัดสมดุล) **ไม่ตัดเป็น "ท่าหล/วงฯ"** ทั้งที่ ~470px และจอแคบกว่า
- ค่าตัวเลขที่แน่นอน (px/rem) จูนตอน implement โดยดูผลใน preview จริง

## การตรวจรับ

- `pytest robot/web/test_server.py` ผ่านทั้งหมด (เทสต์เดิม 10 ตัว + เทสต์ใหม่ของ `wants_map` และ `/ask` แนบรูป)
- เปิด `/screen` แล้ว:
  - ถามคำถามตำแหน่ง (เช่น "สาขาอิเล็กทรอนิกส์อยู่ตึกไหน", "งานพัสดุอยู่อาคารไหน") → คำตอบมีข้อความ **+ รูปแผนผัง** แตะแล้วขยายเต็มจอได้ ปิดได้
  - ถามคำถามทั่วไป (เช่น "ค่าเทอมเท่าไหร่") → **ไม่มี**รูป (เหมือนเดิม)
  - แตะชิป "แผนผังวิทยาลัย" → ขึ้นรูปแผนผังทันที โดยไม่เรียก AI
  - หัวข้อ "วิทยาลัยเทคนิคท่าหลวงฯ" ไม่ตัดคำเสียรูป ทั้งจอกว้าง/แคบ
- เปิด `/face` และ endpoint `/state` → ทำงานเหมือนเดิม (ไม่ถูกแตะ)
- ลองถามชื่ออาคารที่อยู่ใน legend (เช่น "อาคาร 7 มีอะไรบ้าง") → ตอบได้จากฐานข้อมูล; ถามสิ่งที่ไม่มีใน legend → ยังตอบ "ยังไม่มีข้อมูล" ตามกฎกันมั่ว

## ไม่ทำรอบนี้ (เก็บไว้เฟส 2)

- ปักหมุด/ไฮไลต์ตำแหน่งอาคารที่ถามบนรูปแผนผัง (ต้องเก็บพิกัด x/y รายอาคารบน `map.jpg`)
- รูปถ่ายอาคาร/จุดบริการรายจุด (นอกเหนือจากแผนผังรวม)
- มิเรอร์รูปแผนผังไปโชว์บนจอใบหน้า (`/face`)
