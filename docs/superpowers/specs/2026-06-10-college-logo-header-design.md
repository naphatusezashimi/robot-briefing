# Design Spec: โลโก้วิทยาลัยจริงบนหัวจอเนื้อหา

**วันที่:** 2026-06-10
**สถานะ:** อนุมัติแล้ว
**ขอบเขต:** เปลี่ยนไอคอนจำลองบนหัวจอเนื้อหาของเว็บ "กล่องพูดได้" ให้เป็นโลโก้จริงของวิทยาลัย — เพิ่ม asset 1 ไฟล์ + แก้ `screen.html` ไฟล์เดียว (2 จุด: CSS และ HTML)

---

## ที่มา

ทีมได้ไฟล์โลโก้จริงของวิทยาลัยมาแล้ว (`image/1780392362301.jpg` — JPG พื้นขาว ~43KB ตราหกเหลี่ยมแดง-น้ำเงิน "สุ จิ ปุ ลิ" พร้อมริบบิ้นชื่อวิทยาลัยด้านล่าง) แต่หัวจอเนื้อหาปัจจุบันยังใช้กล่องไล่เฉดฟ้า + ไอคอนหมวกบัณฑิต SVG เป็นตัวแทนชั่วคราวอยู่

## การตัดสินใจหลัก (locked)

1. **ตำแหน่ง: หัวจอเนื้อหาเท่านั้น** — จากตัวเลือก 4 จุด (A หัวจอเนื้อหา / B มุมจอใบหน้า / C ลายน้ำกลางแชต / D favicon) ผู้ใช้เลือก A จุดเดียว
2. **กรอบ: ไทล์ขาวขอบมน** — เห็นโลโก้เต็มดวงรวมริบบิ้นชื่อวิทยาลัย
   - ตัด "วงกลม" เพราะมุมริบบิ้นด้านล่างโดนตัด
   - ตัด "ไม่มีกรอบ" เพราะเห็นขอบสี่เหลี่ยมพื้นขาวของ JPG จาง ๆ บนพื้นหัวจอ
3. **วิธีเสิร์ฟไฟล์: คัดลอกเข้า `robot/web/static/`** — เว็บเสิร์ฟโฟลเดอร์นี้อยู่แล้ว ไม่ต้องแก้ server
   - ตัด "เพิ่ม route เสิร์ฟ `image/`" เพราะผูกเว็บกับโฟลเดอร์นอก `robot/` เปราะตอนย้ายขึ้น Raspberry Pi
   - ตัด "ฝัง base64" เพราะ HTML บวม ~58KB แก้ไขยาก
4. **ไม่แตะ:** `server.py`, `face.html`, ไฟล์ `.js` ทั้งหมด, favicon และโฟลเดอร์ `mockups/` (เป็นไฟล์ดีไซน์เก่า ไม่ใช่ตัวเดโม)

## รายละเอียดการแก้

1. **Asset** — คัดลอก `image/1780392362301.jpg` → `robot/web/static/logo.jpg`
   (ต้นฉบับใน `image/` คงไว้เป็น source ไม่ลบ)
2. **`robot/web/static/screen.html`** — แก้ 2 จุด:
   - CSS `.logo`: เปลี่ยนพื้นจากไล่เฉดฟ้า → `#fff`, เพิ่มเส้นขอบ `1px solid #e7eefb` (โทนเดียวกับเส้นแบ่งใต้ header), เงาฟ้าเดิมเปลี่ยนเป็นเงากลาง ๆ จาง ๆ `0 8px 18px -6px rgba(20,60,120,.18)`; ขนาด 48px และ `border-radius:14px` คงเดิม
   - HTML ใน `<div class="logo">`: แทน `<svg>` หมวกบัณฑิต ด้วย
     `<img src="/static/logo.jpg" alt="ตราวิทยาลัยเทคนิคท่าหลวงซิเมนต์ไทยอนุสรณ์" width="40" height="40" style="object-fit:contain">`

## การตรวจรับ

- รัน `robot/web/run_web.bat` → เปิด `/screen` → โลโก้จริงแสดงในไทล์ขาว คมชัด ไม่ยืด/ไม่บีบ
- องค์ประกอบอื่นของ header (ชื่อวิทยาลัย, pill สถานะ) ไม่ขยับเพี้ยน ทั้งที่ความกว้างปกติ (~470px) และจอแคบกว่า
- `/face` และ endpoint อื่นทำงานเหมือนเดิม (ไม่ได้ถูกแตะ)
